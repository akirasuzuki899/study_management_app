## 制作背景
大学三四年生にかけて、TOEICを学習していました。
英語は不得意な上、学科の実験やレポートで忙しかったので、英語学習に割ける時間は限られていました。このような状況で結果を出すためには戦略的に学習する必要があると考え、以下の二点を意識して取り組むことにしました。<br><br>

### 学習の計画<br>
現在のレベルから目標を達成するために必要な学習と時間を調べ、具体的な学習方法を決めました。<br><br>

### 学習の習慣化<br>
いつ何を学習するかを一週間をベースに決めました。そうすることで、「今日は何を勉強しよう」と考える時間を無くし、毎日のルーティーンとしてタスクを処理することで、学習を習慣化しました。<br><br>

テストの結果をもとに、必要であればスケジュールを修正しました。<br>
この二点を徹底したことで、半年で300点から目標スコア700点(現在805点)を達成することができました。しかし次の問題点を解決できれば、より簡単に目標を達成できたと思います。<br><br>


## 解決したい問題
### 1. 週間スケジュールをベースとした学習アプリが欲しい<br>
学習内容を週間スケジュールに落とし込むことで学習を習慣化していたものの、使用していた学習アプリ（abceed, reminDO） はいずれもスケジュール管理機能がありませんでした。その点、smart study はスケジュール機能を持ちますが、以下の問題点があったため使用を見送った経緯があります。<br>
①登録できるタスクが5教科に限定されておりターゲット層が学生である<br>
②反応しないボタンが存在しUXが良くない<br>
③スケジュールに具体的な学習方法を記載するためのコメント欄が存在しない<br><br>
 
以上の問題を解決する学習アプリが欲しいと考えました。<br>
加えて、以下に示す機能を実装したいと考えました。<br>
・予定通りに学習できなかったスケジュールを一目見てわかるように管理し、再登録が行える<br>
・毎週のスケジュールをひとつひとつ作成する手間を省くために、テンプレートから一括で作成できる<br><br>


### 2. 学習の全体像を簡単に把握/整理したい<br>
調査した内容を箇条書きでまとめたり頭の中にとどめておくだけで情報が整理されていない状態でした。そのため、学習の経過を振り返る時に以下の問題点を感じていました。<br>
①調査から時間が経つと、学習の全体像を把握するのに苦労する<br>
②学習の全体像が漠然としているので、目標達成までの距離感を掴みづらい<br>
③目標達成に必要な要素を俯瞰できないので、優先するべきことが見えづらい<br><br>

結果として、無駄に再調査を行う必要がありました。<br><br>

これらの問題を解決する手段として、マンダラチャートは最適だと思います。マンダラチャートとは、9×9の81マスで構成される目標達成ツールです。このツールを使用することで、目標達成に必要なことを視覚的に見やすく整理できます。また、マス目を埋める過程で目標達成に必要な要素と行動が明確になります。<br><br>

マンダラチャートの有名なアプリとして、Grid ToDo Lite があります。高機能で大変使いやすいのですが、1マスごとに作成できるメモ機能については、記入できる文字数が少なく、また複数作成できない点に物足りなさを感じました。<br><br>

目標が大きいほどまとめておきたい事も増えるので、自由度の高いメモ機能が欲しいと感じました。<br><br>

これらの問題点を解決するために、以下に示す機能を実装したいと考えました。<br>
・1マスに対して複数のメモを残せる（リッチテキスト）<br>
・1マスに対して外部リンクを設定できるようにし、情報源に簡単にアクセスできる<br>
・1マスに対して達成状況を設定できる<br><br><br>


## 工夫した点
| 機能 |
| :--- |
* グラフデータの操作を、ORM機能を使わずに生のSQLで記述した（TIMESTAMPDIFF関数、DATE関数、INNER JOIN、GROUP BY の使用）
* スケジュールの特定の時間をクリックすることで、学習予定作成フォームに自動で時間が入力される
* 学習予定をドラッグ&ドロップで更新できる
* テンプレート機能により、学習予定を毎週一つずつ作成する手間を省ける
* 未学習予定を一覧表示することで、学習の進捗状況を簡単に把握できる
* 学習時間が記録されている予定の色を暗めにすることで、未学習予定が一目見てわかる
* スケジュールの表示範囲を一週間ごとに変更するボタンと自由に指定できるカレンダーを実装した
* ディスプレイの横幅が 600px 以上の時、未学習予定の一覧とカレンダーを常時表示できる

| UI/UX |
| :--- |
* レスポンシブデザインに対応し、様々なデバイスで使用できるようにした
（デバイスのサイズによってデザインが崩れないように専用のデザインを用意した）
* 既存アプリの UI/UX を分析し、参考にできる箇所をアプリに取り入れた
* フロントエンドに Vue と Vuetify、バックエンドに Ruby on Rails を用いて完全SPA化し、UXを向上させた
* Vuetify ではできない細かいデザインの調整を css で行い、アプリ全体で統一感のあるデザインにした
* 特定のデータが無い時にユーザーを誘導するボタンを表示<br>
  *  当日の予定がない時&emsp;→&emsp;タイムライン上に週間スケジュールへの誘導ボタンを表示<br>
  *  教材がない時&emsp;&emsp;&emsp;&emsp;→&emsp;タイムライン、教材メモ、教材一覧上に教材登録への誘導ボタンを表示<br>

| インフラ |
| :--- |
 * CircleCI, Amazon ECR, AWS Fargate, Slack を連携し、デプロイの工数を削減して開発効率を向上させた

| その他 |
| :--- |
 * 公式ドキュメントでわからないことがあれば（フレームワークのロジックを詳細に知りたいとき等）、github のソースコードを調査した
 * docker コンテナ内での操作(rspec のテスト実行やデータベースの確認)を shell script で簡略化した
